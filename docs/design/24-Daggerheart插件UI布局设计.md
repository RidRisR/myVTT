# 24 - Daggerheart 插件 UI 布局设计

> **状态**：📋 规划中 | 2026-04-10

## 概述

本文档记录 Daggerheart 插件在 VTT 中的 UI 布局设计决策，涵盖所有面板和组件的视觉规格。设计风格定位为**奇幻/魔法桌游质感**，避免科技/HUD 感。

对应 mockup 文件：

- `nimbalyst-local/mockups/daggerheart-vtt-layout.mockup.html` — 完整界面集成

- `nimbalyst-local/mockups/daggerheart-charcard.mockup.html` — 角色卡详情

## 1. Fear 追踪器

**样式**：浮动药丸（非全宽横条），顶部水平居中
**内容**：`FEAR` 标签 + 12 个圆点 + `n/12` 计数
**视觉**：

- 背景 `rgba(0,0,0,0.3)` + `backdrop-filter: blur(12px)`
- 圆角 20px
- 已填充点：红色 `#e74c3c` + 发光 box-shadow
- 空点：`rgba(255,255,255,0.08)` 边框

## 2. 角色卡（左侧面板）

**定位**：左侧垂直居中，`position: absolute; top: 50%; transform: translateY(-50%)`
**宽度**：220px
**布局**：单栏垂直排列，自适应高度，`max-height: calc(100vh - 100px)`，overflow 滚动
**圆角**：左侧贴边（`border-radius: 0 16px 16px 0`）

### 信息层级（从上到下）

1. **标题行**：「角色卡」+ 收回按钮
2. **角色头部**：头像圆形 32px + 名字 + 职业/等级
3. **属性区（最醒目）**：3×2 网格
   - 每格：中文名 + 英文全称 + 大号数值（14-18px）
   - 正值绿色、负值红色、零值灰色
   - hover 时可点击掷骰（骰子图标提示）
4. **资源区**：
   - 生命 HP：红色条形 + 数值
   - 压力 Stress：紫色条形 + 数值
   - 护甲 Armor：蓝色离散点
   - 希望 Hope：金色离散点
5. **防御阈值**：三格并排（闪避/重伤/严重）
6. **经历列表**：名称 + 修正值，数量不固定
7. **状态区**：标签形式（绿=buff、红=debuff、黄=中性），显示持续回合

## 3. 在场角色列表（右上面板）

**布局**：右侧上半部分，220px 宽，flex: 1 填充
**样式**：圆角面板 `border-radius: 14px 0 0 14px`

### 每行角色

- **迷你头像**（34px）：SVG 弧形显示 HP/Stress 比例
- **信息区**：名字 + 职业提示 + 四项微型数值（HP/Stress/Armor/Hope）+ 状态标签
- **玩家/敌方分区**：中间分隔线 + `敌方` 标签
- **敌方特殊样式**：红色头像边框 + `敌` 角标，只显示 HP/Stress

### Hover 弹出卡（待实现）

- 弹出方向：向左
- 内容：全名+职业/等级、HP/Stress 条形、Armor/Hope 点形、6 属性、状态列表+持续时间

## 4. 聊天面板（右下）

**布局**：右侧下半部分，与角色列表同宽
**固定高度**：170px
**内容**：消息列表 + 输入框
**样式**：圆角 `14px 0 0 14px`，与角色列表对齐

---

## 5. 底部头像（当前角色）

**定位**：底部水平居中
**整体尺寸**：约 130×130px

### 五层同心结构（从外到内）

1. **符文环 + Hope 宝石**（最外层）
   - 符文环：126px，缓慢旋转（80s），ogham 字符装饰
   - Hope 宝石：6 颗金色宝石嵌在上弧，填充时有径向渐变 + 闪烁动画

2. **HP/Stress 弧形**（110px）
   - HP（红色）：左半圆，从底部向上延伸
   - Stress（紫色）：右半圆，从底部向上延伸
   - 带发光滤镜 + 脉冲动画

3. **护甲板环**（82px）
   - SVG `stroke-dasharray` 实现动态分槽
   - 每块板弧度 = 周长 / 总槽数 - 间隙
   - 支持 3-12 槽动态调整
   - 损坏槽用半透明黑色覆盖层

4. **头像核心**（52px）
   - 圆形头像，金色边框 + 发光

5. **昵称**（底部）

## 数据字段参考

从 `.st` 文件可导入的字段：

| 类别        | 字段                                                                       |
| ----------- | -------------------------------------------------------------------------- |
| 属性（6项） | 敏捷、力量、本能、知识、风度、灵巧                                         |
| 资源        | 生命(current/max)、压力(current/max)、希望(current/max)、护甲(current/max) |
| 防御        | 闪避、重伤阈值、严重阈值                                                   |
| 经历        | name + modifier 列表（数量不固定）                                         |

## 设计原则

1. **奇幻桌游质感**：符文、宝石、暖色点缀，避免科技/HUD 直线风格
2. **信息密度适中**：角色卡属性优先（最常查看），资源次之
3. **交互提示**：属性格子 hover 显示骰子图标，点击即可掷骰
4. **响应式高度**：角色卡自适应内容，不强制撑满；经历多时面板可滚动
5. **画布空间优先**：所有面板靠边贴合，中央留最大画布给地图
