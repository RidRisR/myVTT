## 1. 场景与背景图解耦

**问题**: 当前"场景 = 一张背景图"，创建场景就是上传图片，切换场景就是换背景。但场景应该是一个容器，包含多种属性。

**决策**: 场景作为容器，氛围图和战斗地图是其中两个独立属性。

**场景包含的属性**:

| 属性       | 说明                                       | 必填               |
| ---------- | ------------------------------------------ | ------------------ |
| 名称       | 场景标题（如"黑暗森林"）                   | 是                 |
| 氛围图     | 叙事模式全屏背景                           | 否（可以是空场景） |
| 战斗地图   | 战术模式用的地图图片（不设置则复用氛围图） | 否                 |
| 粒子特效   | 预设选择（余烬/飘雪/萤火虫等）             | 否，默认无         |
| 网格配置   | gridSize/offset/snap/color                 | 否，有默认值       |
| Token 数据 | 场景内的 Token                             | 自动管理           |
| 场景实体   | 场景专属 NPC                               | 自动管理           |
| 音乐/音效  | 背景音乐 URL                               | 否，后续扩展       |

**核心变化**: 叙事模式显示氛围图，战术模式可以使用另一张带网格的战斗地图。两张图独立上传、独立管理。

## 2. 场景管理界面

**决策**: 提供集中的场景管理入口。

**场景列表面板**（从 GM 工具栏打开）:

- 显示所有场景的缩略图 + 名称
- 点击场景 → 切换为活跃场景
- 每个场景有编辑按钮 → 打开场景配置面板
- 支持新建/删除场景

**场景配置面板**（编辑单个场景）:

- 场景名称
- 氛围图：上传/更换/清除
- 战斗地图：上传/更换/清除（可选）
- 粒子特效：下拉选择预设
- 网格配置（关联战斗地图）
- 音乐 URL（后续扩展，先留位置）

**创建场景的两种路径**:

- **完整路径**: 新建空场景 → 逐项配置属性
- **快捷路径**: 上传图片 → 自动创建场景并设为氛围图（兼顾轻备团）

## 3. 叙事模式 UI 布局

**决策**: 确认各 UI 元素在叙事模式下的位置。

```
┌─────────────────────────────────────────────────┐
│  [HamburgerMenu]   [PortraitBar 顶部居中]        │
│                                                   │
│                                                   │
│                   全屏氛围图                       │
│                 + 粒子特效层                       │
│                                                   │
│                                                   │
│  ┌──────────┐                    ┌──────────┐    │
│  │ 角色卡    │                    │ ChatPanel │    │
│  │ (左下)    │                    │ (右下)    │    │
│  └──────────┘                    └──────────┘    │
│                                                   │
│           [GM 工具栏 — 底部中央]                    │
│              [BottomDock — 按需展开]               │
└─────────────────────────────────────────────────┘
```

**注**: 角色卡和聊天面板的左右位置保持当前实现。

| 元素            | 位置                       | 可见性           |
| --------------- | -------------------------- | ---------------- |
| PortraitBar     | 顶部居中                   | 始终可见         |
| 氛围图 + 粒子   | 全屏背景                   | 始终可见         |
| MyCharacterCard | 左下或右下                 | 可展开/折叠      |
| ChatPanel       | 左下或右下（与角色卡对称） | 可展开/折叠      |
| GM 工具栏       | 底部中央                   | 仅 GM 可见       |
| BottomDock      | 底部，按需展开             | 按需             |
| HamburgerMenu   | 角落                       | 始终可见（图标） |
| ShowcaseOverlay | 全屏覆盖                   | GM 推送时出现    |

## 4. GM 工具栏（叙事模式）

**决策**: 底部精简工具栏，4-5 个按钮。

| 按钮         | 功能                 |
| ------------ | -------------------- |
| 场景管理     | 打开场景列表面板     |
| 开启战术地图 | 一键切入战术模式     |
| 素材库       | 展开/收起 BottomDock |
| 推送展示材料 | Showcase 功能        |

**设计原则**: GM 最高频操作是切换场景和开关战术地图，必须一键触达。工具栏尽量精简，不信息过载。
