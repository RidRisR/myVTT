## 已完成的核心决策摘要

### 场景系统

1. **场景 = 内容预设单元**，不是叙事必须切换的单位。GM 可一个场景走完全程
2. **场景内随时换背景**，不需要切场景
3. **新房间自动创建默认场景**，内容为空，PC 自动加入

### 数据拓扑

4. **全局唯一 存储**：所有 Entity 统一存储在 entities（唯一数据源），无 scene.entities
5. **场景持有 entityIds**：per-scene 引用列表，PC 自动加入可移除，NPC 创建时加入
6. **删场景 GC**：遍历清理无引用的非 PC Entity（不用引用计数，实时扫描）

### 创建模式

7. **Unlinked（默认）**：从 blueprint 创建 → 新 Entity + 加入当前场景
8. **Linked**：引入已有 Entity → 只加 entityIds（跨场景共享状态）
9. **无 Promote**：所有 Entity 天生在 entities，不需要提升机制

### 战斗模式

10. **战斗是场景内部状态**（scene.combatActive），不是房间级模式
11. **battleMapUrl 可选**：战斗地图可以和叙事背景不同
12. **room.mode 和 room.combatSceneId 废弃**

### 场景切换

13. **场景间不继承** tokens/entityIds，只有全局数据（entities、chat、seats）自动共享

### UI

14. **Token 孤儿降级**为无名棋子
15. **Portrait Bar 分区**：PC 区 + NPC 区（可折叠）

### 权限与持久性

16. **不加 PC/NPC type 字段**，行为差异几乎全由权限决定
17. **persistent 标记**：`persistent: boolean` 控制自动加入所有场景 + GC 保护，PC 默认 true
18. **权限 UI 嵌入角色卡**，不做独立面板

### 可见性

19. **Token 与 Entity 统一权限模型**：废弃 `gmOnly`，Token 自带 `permissions` 字段（同 Entity 结构）
20. **有 Entity 的 Token** 用 Entity permissions，**无 Entity 的 Token** 用自身 permissions

### 待实现

21. **场景重置**（快照 save/load）— 优先级不高，预留数据结构
22. **素材库标签系统** — 全局素材，标签组织
23. **Linked 放置 UI** — 跨场景复用已有角色的交互方式

## 贯穿全局的设计哲学

> **快速创建 + 渐进丰富**：GM 应该能以最少的操作把东西放到舞台上，然后按需添加细节。系统不应该要求用户在创建时做出架构层面的决策。
