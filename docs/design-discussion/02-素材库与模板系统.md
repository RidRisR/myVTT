# 议题 2: 素材库与模板系统

> 讨论蓝图(Blueprint)、素材库的定位与 UI 呈现。

---

## 已确认决策

### S1: 用户不需要知道"图片"与"模板"的区别

GM 的心智模型：**"我有一堆素材，想把它放上去"**。系统不应要求 GM 在上传时选择"这是纯图片还是带数据的模板"。

- 纯图片和带数据模板在 UI 上**统一呈现**
- 有数据的素材可用**次要视觉线索**（小角标等）提示，但不做硬分区
- 点击行为一致：纯图片创建空 Entity，带数据的预填属性

### S2: 渐进丰富闭环

```
上传图片 → 纯图片素材（零门槛）
         ↓ 放上舞台 → 创建空 Entity
         ↓ GM 在角色卡里配属性
         ↓ "保存为模板" → 数据回写到素材
         ↓ 下次使用时自动带属性
```

GM 永远不需要提前决定"创建图片还是模板"。

### S3: 素材命名策略

- 上传时默认使用**文件名**（去掉扩展名）
- 已有**双击改名**功能
- 无意义文件名（如 IMG_2847）不需要系统干预——文件名 + 可编辑已经够用
- 不需要随机名/自动命名机制

### S4: 底栏 Tab 结构维持现状

当前三个 tab（Maps / Tokens / Handouts）按**用途**分类，方向正确：

- Maps → 场景背景/战斗地图
- Tokens → 角色素材（纯图片 + 带数据模板混合）
- Handouts → 给玩家展示的文档/参考图

暂不做合并或重命名，等后续痛点出现再调整。

### S5: 标签系统 — 纯自由标签

当前素材量（< 30）不需要标签。未来需要时采用**纯自由标签**：

- GM 自由输入标签文字（如 `兽人`、`酒馆`、`boss`）
- 不做预设分类维度——myVTT 不绑定特定规则系统，D&D 和 CoC 的分类完全不同
- 数据结构：`tags: string[]`
- 输入方式：素材项编辑时输入，逗号分隔或 tag input 组件
- 优先级低，等素材量增长到造成实际困惑时再实现

### S6: "保存为模板" — 显式操作

模板和实例必须解耦：修改舞台上的 Entity 不应自动改动素材库的 Blueprint。

- 触发方式：角色卡右键菜单 →"保存为模板"
- 有 blueprintId → 询问覆盖还是另存新模板
- 无 blueprintId → 创建新 Blueprint，回写 blueprintId 到 Entity
- 回写内容：ruleData、size、color 等默认值
- 不做自动回写（同一 Blueprint 可能生成多个实例，各自独立）

### S7: 存储保持分离，UI 统一入口

场景、角色素材、文档素材在 Y.Map 层**保持分开**：

| 存储                   | 内容                             | 理由                             |
| ---------------------- | -------------------------------- | -------------------------------- |
| `scenes` Y.Map         | 场景（含运行时 entities/tokens） | 场景是运行时容器，不是纯素材     |
| `blueprints` Y.Map     | 角色素材/模板                    | 纯素材，生命周期简单             |
| `handout_assets` Y.Map | 文档/参考素材                    | 有特殊字段（文字内容、展示模式） |

- 不强行合并——场景有运行时状态（entities、tokens、grid），跟纯素材本质不同
- UI 层已用 tab 做分类呈现，底层是否统一对 GM 透明
- blueprints 和 handout_assets 未来可考虑合并，但不急

## 数据层方向（记录但不急于实现）

Blueprint 类型可在未来演进为更通用的 AssetEntry：

```typescript
// 当前
interface Blueprint {
  id: string
  name: string
  imageUrl: string
  defaultSize: number
  defaultColor: string
  defaultRuleData?: unknown
}

// 未来可能的演进
interface AssetEntry {
  id: string
  type: 'character' | 'scene' | 'handout'
  name: string
  imageUrl: string
  tags: string[]
  defaultSize?: number
  defaultColor?: string
  defaultRuleData?: unknown
}
```

这个演进不紧急，当前 Blueprint 类型够用。
