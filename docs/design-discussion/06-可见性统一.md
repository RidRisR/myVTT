# 议题 6: Token 与 Entity 可见性统一

> 讨论 Token 和 Entity 的可见性控制是否应统一。

---

## 已确认决策

### V1: 统一权限模型

Token 和 Entity 使用**同一套权限结构** `EntityPermissions`，三种状态：

| 权限       | Portrait Bar | Token  | 可操作 |
| ---------- | ------------ | ------ | ------ |
| `none`     | 不可见       | 不可见 | 否     |
| `observer` | 可见         | 可见   | 否     |
| `owner`    | 可见         | 可见   | 是     |

### V2: Token 自带 permissions 字段

废弃 `token.gmOnly: boolean`，替换为 `token.permissions: EntityPermissions`。

```typescript
// 旧
interface MapToken {
  gmOnly: boolean       // ← 废弃
  entityId?: string
  ...
}

// 新
interface MapToken {
  permissions: EntityPermissions   // ← 与 Entity 同结构
  entityId?: string
  ...
}
```

### V3: 可见性解析规则

```
Token 可见性：
  有 entityId → 用 Entity 的 permissions（Token 自身 permissions 忽略）
  无 entityId → 用 Token 自身的 permissions
```

- 一条代码路径：`canSee(effectivePermissions, seatId, role)`
- 无特殊分支，无 `if (gmOnly)` 逻辑

### V4: 不做 Portrait/Token 独立可见性

不支持"portrait 可见但 token 隐藏"的场景。先统一，未来有需求再拆分。

设计哲学：**先统一再增加 > 先零散再统一**。
