## 1. CSS 方案 → Tailwind

**背景**: 项目当前 100% 内联样式（React.CSSProperties），38 个文件中共 447 处 `style={}`。

**问题**:

- 内联样式无法实现 `:hover`、`:focus-visible` 伪类
- 内联样式无法实现 `@media (prefers-reduced-motion)` 媒体查询（WCAG 硬性要求）
- 内联样式无法实现 `@keyframes` 动画（当前用 JS 注入 `<style>` 标签绕过）
- 颜色魔数散落在 30+ 组件中，无法全局调整

**决策**: 迁移到 Tailwind CSS。

**理由**:

- Tailwind 的 class 直接写在 JSX 里，和内联样式的心智模型一致——不需要写单独的 CSS 文件
- 通过 class 前缀原生支持 `hover:`、`focus-visible:`、`motion-reduce:`、响应式断点
- Tailwind config 本身就是设计 Token 系统
- 迁移可结合后续 UI 重构逐步进行，不需要一次性替换全部 447 处
- 动态值（运行时计算的位置、用户选择的颜色等）仍保留 `style={}`

**迁移策略**: 后续 UI 重构时，逐组件迁移内联样式到 Tailwind class。

## 4. 动画/过渡统一规范

**Duration 三档**:

| Token    | 值    | 用途                                |
| -------- | ----- | ----------------------------------- |
| `fast`   | 150ms | hover 变色、按钮按下、开关切换      |
| `normal` | 250ms | 面板弹入/弹出、下拉展开、Tab 切换   |
| `slow`   | 400ms | 场景切换、ShowcaseOverlay、全屏过渡 |

**Easing 原则**:

- 元素进入 → `ease-out`（快速响应 → 优雅到位）
- 元素退出 → `ease-in`（缓慢启动 → 快速离开）
- Tailwind 内置 `ease-in`/`ease-out`/`ease-in-out` 基本够用

**属性原则**:

- 禁用 `transition: all`，必须显式列出要动画的属性
- Tailwind 里写 `transition-opacity`、`transition-transform`、`transition-colors`

**`prefers-reduced-motion` 处理**:

- 使用 Tailwind 的 `motion-reduce:` 前缀
- 示例: `transition-opacity duration-normal motion-reduce:duration-0`

## 5. SVG 图标 → Lucide React

**背景**: 当前 30+ 组件各自内联 `<svg>`，尺寸（14/16/18）和 strokeWidth（1.5/2/2.5）不统一。

**决策**: 引入 Lucide React 图标库。

**理由**:

- 700+ 开源图标，统一 24×24 viewBox
- Tree-shakeable，只打包使用的图标
- 用法简洁: `<Menu size={16} />`

**迁移策略**: 组件重构时逐步替换内联 SVG，不需要一次性迁移。

## 6. color-scheme: dark + meta theme-color

**决策**: 在 `index.html` 中添加:

- `<html style="color-scheme: dark">` — 让原生滚动条、表单控件自动适配深色
- `<meta name="theme-color" content="#0f0f19">` — 让移动端浏览器标题栏匹配主题

**实施**: 一次性添加，无争议。
