## 1. 场景准备流程 — 分布式就地配置

**决策**: 不使用"创建场景向导"，所有配置分散在各自的自然入口处完成。

**理由**: 向导会强制 GM 一次性决定所有配置，违反"轻备团优先"原则。分布式配置允许 GM 按需逐步完善。

**各配置项的入口**:

| 配置项         | 入口位置                           |
| -------------- | ---------------------------------- |
| 氛围图/背景    | 底部素材库 Maps 标签页上传/选择    |
| 战斗地图       | 底部素材库 Maps 标签页，关联到场景 |
| 网格设置       | 战术地图打开后，战术工具条中配置   |
| NPC/Token 预设 | 战术地图上右键创建或从蓝图拖入     |

**备注**: 网格配置（间距、偏移、对齐）需要可视化工具辅助，纯数值输入不够直观。具体设计在议题 15（战斗地图工具）中细化。

## 2. 战术地图形态 — 浮动面板

**决策**: 战术地图以**浮动面板**形式叠加在氛围图上方，不替换氛围图（不切屏）。

**核心行为**:

- GM 一键打开 → 战术地图浮现在氛围图上方
- GM 一键收起 → 战术地图消失，回到叙事模式
- Token 数据保留，收起后隐藏，再次打开恢复
- 面板可缩放/拖动（具体交互待实现时调整）

**设计参考**: Alchemy RPG 的 Tactical Mode — "Tactical Mode sits on top of Story Mode"，隐藏战术地图即回到氛围画面。

**玩家端**: 战术地图的打开/收起由 GM 控制，所有客户端同步。

## 3. NPC 快速创建 — 四种场景

**决策**: 覆盖 4 种常见 NPC 创建场景，批量放置采用"一个一个放 + 快速复制"。

| 场景           | 操作方式                                                                 |
| -------------- | ------------------------------------------------------------------------ |
| **临时创建**   | 战术地图空白处右键 → 立即创建无名 Token（默认圆形 + 颜色），之后补充细节 |
| **从蓝图放置** | 从底部素材库 Tokens 标签页拖入地图（或从 PortraitBar 拖入）              |
| **批量同类**   | 先放置一个，然后通过右键菜单"复制 Token"快速创建副本                     |
| **场景预设**   | GM 提前在战术地图上摆好 Token，收起地图隐藏。战斗开始时一键展开          |

**不采用"批量放置模式"**（如连续点击地图放多个），保持操作简单。

## 4. 先攻系统 — 复用 PortraitBar

**决策**: 不新增 UI 元素，复用现有 PortraitBar 兼做先攻条。

**两种模式下的行为差异**:

| 模式     | PortraitBar 行为                                                   |
| -------- | ------------------------------------------------------------------ |
| 叙事模式 | 显示当前场景的角色头像，展示/选择用途                              |
| 战术模式 | 切换为先攻条：按先攻顺序排列，当前行动角色高亮，支持"下一回合"推进 |

**骨架 vs 规则层**:

- **骨架提供**: 手动拖拽排序、当前角色高亮、"下一回合"按钮推进
- **规则层接口**: `setInitiativeOrder(entityIds[])` — 规则插件可自动计算先攻值并设定顺序
- 骨架不关心先攻值怎么来的，只负责展示和手动调整

## 5. PortraitBar 拖放到地图

**决策**: 支持从 PortraitBar 直接拖拽角色头像到战术地图上，在落点位置创建对应 Token。

**理由**: 比"打开素材库 → 找到蓝图 → 拖入"路径更短，符合"直觉式交互"原则。

**实现要点**: 拖拽过程中显示拖影（drag ghost），落点显示网格吸附预览。

## 6. 场景切换行为

**决策**: 切换场景时**自动收起战术地图**，先回到叙事模式，再执行场景过渡。

**完整流程**: 战术地图收起动画 → 氛围图 crossfade 过渡 → 新场景以叙事模式呈现。

**理由**: 场景切换是叙事事件（"你们来到了新的地方"），应先让玩家感受新场景氛围，而非直接跳到另一张战斗地图。GM 如需在新场景开战，再手动打开战术地图。

**备注**: 具体过渡动画效果待实现时调整。

## 7. GM 笔记 ⏸

**决策**: 推迟实现。后续如果 UI 有空间，加一个按钮弹出面板即可。

**当前状态**:

- 实体级笔记：现有 `notes` 字段输入框太小，不实用 → 在议题 14（角色与 Token 交互）中重新设计
- 场景级 GM 笔记：不急，简单功能（按钮 + 面板），优先级不高

## 8. GM 预览玩家视角

**决策**: 仅在战术模式下提供，战术工具条加一个"玩家视角"切换按钮。

**行为**:

- 按下后：GM 看到的内容与玩家一致（隐藏 GM-only Token、隐藏 GM 标记等）
- 再按一下：切回 GM 视角

**叙事模式不需要**: 叙事模式下 GM 和玩家看到的内容基本一致（同一张氛围图）。

## 9. GM 工具栏布局

**决策**: 两套工具栏，按模式独立存在。

| 工具栏         | 位置                     | 包含内容                                       | 何时可见           |
| -------------- | ------------------------ | ---------------------------------------------- | ------------------ |
| **叙事工具栏** | 屏幕底部                 | 切换场景、开启战术地图、推送展示材料、管理角色 | 始终可见（仅 GM）  |
| **战术工具条** | 战术地图面板边缘（竖向） | 下一回合、玩家视角、网格开关、收起地图         | 战术地图打开时可见 |

**关键**: 战术工具条与战术地图面板是一体的，收起地图时工具条一起消失。

**备注**: 当前底部工具栏的视觉设计比较随意，后续在议题 22（视觉风格设计）中统一美化。
