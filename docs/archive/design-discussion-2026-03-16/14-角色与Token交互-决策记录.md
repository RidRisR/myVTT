## 1. 资源条拖动交互 — Awareness 实时广播

**决策**: 资源条（如 HP）支持直接拖动修改，拖动过程通过 Yjs Awareness 实时广播给其他客户端。

**交互流程**:

- 拖动过程中：更新本地 React 状态（60fps 流畅）+ 通过 Awareness 广播当前值（~20fps 节流）
- 拖动结束（pointerUp）：写入 Yjs（持久化 + 网络同步）
- 其他客户端：从 Awareness 读取中间值，实时显示拖动动画

**Token 拖动同理**: 拖动时本地 + Awareness 广播位置，释放时 Yjs 写入。其他客户端看到 Token 的移动过程。

## 2. 并发编辑冲突 — Awareness 软锁

**决策**: 当前仅在视觉层面防止并发编辑冲突，不做底层冲突解决机制。

**Awareness 软锁机制**:

- 开始编辑/拖动时：通过 Awareness 广播 `{ editing: { targetId, field } }`
- 其他客户端：看到该字段正在被编辑 → 显示视觉指示（如高亮/锁定图标）→ 阻止用户操作该字段
- 编辑结束/断线：Awareness 自动清除 → 锁解除

**暂不考虑的底层方案**: UndoManager、Y.Text 字符级合并等细粒度冲突解决，等实际遇到问题再引入。

## 3. Y.Text vs Y.Map 值

**决策**: 文本类字段使用 Y.Text，数值类字段使用 Y.Map 简单值。

| 字段类型            | 存储方式 | 原因                                                                          |
| ------------------- | -------- | ----------------------------------------------------------------------------- |
| 笔记（notes）       | Y.Text   | 可能多人同时编辑，需要字符级 CRDT                                             |
| 名称（name）        | Y.Text   | 同上                                                                          |
| 描述（description） | Y.Text   | 同上                                                                          |
| HP/属性/数值        | Y.Map 值 | 数值不可拆分为字符级编辑（"15"和"10"合并会变成乱码），用 Awareness 软锁防并发 |
| 位置（x, y）        | Y.Map 值 | 同数值                                                                        |

## 4. Token 右键菜单

**决策**: Token 和空白区域分别弹出不同的右键菜单。

**右键 Token**:

- 复制 Token
- 删除 Token
- 属性（打开详情面板）
- 尺寸切换（1×1 / 2×2 / 3×3 / 4×4）
- 可见性切换（GM 隐藏/显示）

**右键空白处**:

- 立即创建无名 Token（默认圆形 + 随机颜色，先创建后完善）

## 5. Token 悬浮提示

**决策**: 鼠标悬浮 Token 时显示轻量提示信息。

**提示内容**:

- Token 名称
- HP 迷你条（彩色进度条）
- 状态图标（如中毒、昏迷等，来自 statuses 字段）

**行为**: 悬浮约 300ms 后显示，鼠标移开立即消失。拖动过程中不显示。

## 6. Token 层叠 ⏸

**决策**: 推迟处理。当前不处理多 Token 重叠时的选择问题，由用户自行避免重叠。

## 7. Token 尺寸 ⏸

**决策**: 开发时细化。基本方向：size 字段控制占格数（1/2/3/4），渲染时覆盖对应格子。

## 8. 实体搜索 ⏸

**决策**: 开发时细化。基本方向：蓝图库和地图库顶部加搜索框，前端即时过滤。
