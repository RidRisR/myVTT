## 1. 破坏性操作 — 混合确认策略

**按操作严重程度区分**:

| 操作           | 确认方式               | 理由                                        |
| -------------- | ---------------------- | ------------------------------------------- |
| 删除场景       | 确认对话框             | 影响大 + 有级联 GC（会连带清理孤儿 entity） |
| 删除角色       | 确认对话框             | 影响大 + 可能关联多场景                     |
| 清空聊天记录   | 确认对话框             | 不可逆、数据量大                            |
| 删除单个 Token | Undo Toast（5 秒窗口） | 频繁操作、影响可控、无级联                  |

**关于 Undo 的分布式安全性**:

- 单体删除的 Undo 是安全的：被删除的东西不会被其他客户端并发修改
- 级联删除（场景删除触发 entity GC）不适合 Undo：`deleteScene` 和 `gcOrphanedEntities` 目前在两个独立事务中，UndoManager 可能无法完整回滚
- **实施注意**: 如果将来想给场景删除也加 Undo 能力，需要把两个操作合进同一个 `yDoc.transact()`

## 2. Toast 通知系统

**类型与行为**:

| 类型 | 用途                     | 左边条颜色     | 自动消失         |
| ---- | ------------------------ | -------------- | ---------------- |
| 成功 | 上传完成、保存成功       | 绿色 (success) | 3 秒             |
| 错误 | 上传失败、连接断开       | 红色 (danger)  | 不消失，手动关闭 |
| 警告 | 权限不足、接近限制       | 黄色 (warning) | 不消失，手动关闭 |
| 信息 | Token 已添加、场景已切换 | 蓝色 (accent)  | 3 秒             |
| Undo | 已删除 xxx [撤销]        | 蓝色 (accent)  | 5 秒             |

**位置**: 待定（建议右上角偏下或顶部居中，避免遮挡四角 UI），多个 Toast 从上往下堆叠。

**无障碍**: Toast 区域使用 `aria-live="polite"`。

**与现有 ChatPanel ToastStack 的关系**: 独立系统，ChatPanel 的消息 Toast 保持现有逻辑不变。

## 3. 加载状态

| 场景               | 方案                         | 说明                    |
| ------------------ | ---------------------------- | ----------------------- |
| 图片上传           | 按钮内 spinner + 禁用点击    | 防止重复提交            |
| WebSocket 初次连接 | 全屏居中 spinner + "连接中…" | 连接成功前无法操作      |
| 场景切换           | 保持现有 crossfade           | 已有过渡效果            |
| 素材库首次加载     | 简单 spinner                 | 数据来自 Yjs 本地，极快 |

**文案规范**: 加载提示使用省略号字符 `…`（U+2026），不使用三个点 `...`。

## 4. WebSocket 连接状态

**决策**: 断线时**全屏遮罩阻断操作**，不允许用户继续操作前端。

**理由**: VTT 是"所有人必须看到同一个画面"的应用。断线时继续操作会导致重连后数据冲突混乱。阻断操作比允许离线编辑更安全。

**三种状态**:

| 状态     | 视觉                                 | 行为           |
| -------- | ------------------------------------ | -------------- |
| 已连接   | 无指示                               | 正常操作       |
| 重连中   | 半透明遮罩 + spinner + "重新连接中…" | 所有操作被阻断 |
| 持续离线 | 遮罩 + "连接已断开" + 重试按钮       | 手动重试       |

**实现**: 监听 WebsocketProvider 的 `status` 事件（`'connected'`/`'disconnected'`），控制遮罩显隐。

## 5. 空状态引导

**原则**: 空状态不只是"告诉用户这里空了"，而是提供一个立即可操作的入口（按钮/拖放区域）。有内容后引导消失。

| 位置                | 空状态触发条件 | 引导内容                  |
| ------------------- | -------------- | ------------------------- |
| 主画面              | 没有活跃场景   | "上传第一个场景背景" 按钮 |
| PortraitBar         | 当前场景无角色 | "添加角色" 按钮/提示      |
| BottomDock - Maps   | 无场景         | "上传地图" 拖放区域       |
| BottomDock - Tokens | 无蓝图         | "上传 Token 图片" 提示    |
| ChatPanel           | 无消息         | 灰色提示 "说点什么吧…"    |

## 6. Error Boundary

**决策**: 必须添加。每个独立功能区域包裹一个 Error Boundary。

**建议放置位置**:

- SceneViewer / CombatViewer
- PortraitBar
- MyCharacterCard
- ChatPanel
- BottomDock
- ShowcaseOverlay

**崩溃后 UI**: 显示 "出了点问题" + 重试按钮，不影响其他区域。

## 7. 键盘快捷键体系 ⏸

**决策**: 推迟至议题 13（GM 工作流）、14（Token 交互）、15（地图工具）讨论完之后统一编排。

**原因**: 快捷键设计依赖于"有哪些操作"，当前操作集还未完全确定。
