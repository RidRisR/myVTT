# 议题 4: 权限模型细化

> 讨论权限修改 UI、observer 粒度、PC/NPC 区分方式、persistent 标记。

---

## 已确认决策

### P1: 现有权限结构保持不变

当前权限模型已经满足需求：

```typescript
interface EntityPermissions {
  default: PermissionLevel // 'none' | 'observer' | 'owner'
  seats: Record<string, PermissionLevel>
}
```

- `getPermission(entity, seatId)` → seats 优先，fallback 到 default
- `canSee` → GM 始终 true，玩家检查非 'none'
- `canEdit` → GM 始终 true，玩家检查 'owner'
- `canDragToken` → 同 canEdit 逻辑

### P2: "开明 GM" 场景已支持

GM 想让所有玩家都能拖动所有 token → 将相关 Entity 的 `default` 设为 `'owner'`。无需新机制。

### P3: PC/NPC 不加 type 字段

PC 和 NPC 在数据层没有本质区别，大部分行为差异来自权限：

| 行为            | 来源                          |
| --------------- | ----------------------------- |
| 玩家能编辑/拖动 | `permissions.seats` 有 owner  |
| 玩家能看到      | `permissions.default` 非 none |
| GM 始终可控     | role === 'gm' 硬编码          |

唯二不能从权限推导的行为（自动加入所有场景、GC 保护）通过 `persistent` 标记解决。

### P4: persistent 标记

新增 `persistent: boolean` 字段到 Entity：

- `persistent: true` → 自动加入所有场景的 entityIds（包括未来新建的）+ 删场景时 GC 保护
- `persistent: false`（默认）→ 只出现在被显式添加的场景
- PC 创建时默认 `persistent: true`
- GM 可随时切换（如：NPC 狗被领养 → 打开 persistent）
- 即使 persistent，GM 也可从单个场景手动移除

**验证场景（狗狗领养）：**

```
1. 狗狗作为 NPC 创建在酒馆场景（persistent: false）
2. 玩家领养 → GM 操作：
   a. 给玩家 owner 权限（玩家可控制它）
   b. 打开 persistent（自动出现在所有场景）
3. 切到任何新场景，狗狗都在
```

### P5: Observer 粒度 — 暂不细化

当前 observer = 可看不可编辑，满足需求。未来如需更细粒度（如：能看 HP 但不能看笔记），再扩展 PermissionLevel。

### P6: 权限修改 UI — 嵌入角色卡

- 不做独立权限面板
- 在角色卡里提供轻量入口（小图标或菜单项）
- 包含：owner 指派、默认可见性、persistent 开关
- 具体 UI 形态实现时再定
