# 场景模式 UI 布局设计

> **注意**：本文档中关于 GmToolbar / BottomDock 的设计**取代** `2026-03-12-tactical-map-immersive-design.md` 中对应部分的描述。

## 背景与动机

当前场景模式的 UI 布局是迭代开发中逐步累积的产物，缺乏系统性的整体规划。各组件（GM 工具栏、底部 Dock、角色卡、聊天面板等）的位置和层次关系需要重新梳理，以实现：

1. **氛围沉浸优先**（Alchemy RPG 风格）——场景背景应尽可能占据视野
2. **所有 UI 均可收起**——最大化沉浸感
3. **GM 与玩家视角区分**——GM 有更多工具，玩家界面更简洁
4. **统一的设计语言**——暗色暖调，烛光羊皮纸美感

## 各组件详细设计

### ① 汉堡菜单（不变）

- **位置**：`fixed top-3 left-4 z-toast`
- **内容**：座位设置、头像上传、颜色选择、主题切换、离开座位
- 无变更

### ② 肖像栏（新增可收起）

- **位置**：`fixed top-3` 水平居中
- **现有功能保持**：PC/NPC 头像、资源环、悬浮预览卡、先攻序 Tab
- **新增**：收起按钮/手柄，收起后隐藏整个肖像栏

### ③ 团队面板（新增可收起）

- **位置**：`fixed top-3 right-4`
- **现有功能保持**：全队 HP/资源追踪器
- **新增**：收起按钮，收起后隐藏面板

### ④ 角色卡抽屉（不变）

- **位置**：左侧居中，`fixed` 定位
- **现有功能保持**：头像、名称、4 Tab（资源/属性/状态/笔记）、收起手柄
- 已有收起功能，无需变更

### ⑤ 场景按钮（改进）

- **位置**：`fixed bottom-3 left-4`，与 GM 工具栏分离
- **交互方式**：点击弹出场景列表面板（保持现有模式）

**场景列表面板改进：**

- 场景卡片改为**大图模式**：背景图全宽显示（宽度 100%，高度约 64px）
- 场景名称叠加在底部渐变层上
- 当前场景高亮（金色边框 + 完全不透明），其他场景半透明
- 每个场景卡片保留编辑按钮（✏️），新增**复制按钮**
- 列表末尾新增**空白卡片 + 加号**，点击创建新场景

**战斗按钮从此处移除**，移至 GM 工具栏。

### ⑥ GM 工具栏（新组件）

**这是本次设计中变化最大的组件**，将原来的 `GmToolbar`（左下 2 个按钮）和 `BottomDock`（底部居中 3 Tab）合并为一个统一的多 Tab 组件。

- **位置**：`fixed bottom-3` 水平居中
- **仅 GM 可见**

**Tab 结构：**

| Tab      | 图标 | 内容                                      | 来源                          |
| -------- | ---- | ----------------------------------------- | ----------------------------- |
| 📁 图库  | 📁   | 所有图片资产（背景图 + 战术地图统一管理） | 原 BottomDock 的 Maps Tab     |
| 👤 Token | 👤   | Token 蓝图管理、Spawn NPC                 | 原 BottomDock 的 Tokens Tab   |
| 📜 讲义  | 📜   | 讲义资产管理、Showcase 推送               | 原 BottomDock 的 Handouts Tab |
| 🎲 掷骰  | 🎲   | 掷骰快捷按钮（待设计，先渲染占位 Tab）    | 新增                          |

**固定按钮（不属于任何 Tab）：**

- ⚔️ 战斗切换：位于 Tab 栏右侧，用分隔线与 Tab 区分
- 点击开/关当前场景的 `combatActive` 状态
- 战斗中高亮显示（danger 色）

**图库 Tab 设计要点：**

- **不区分背景图和战术地图**——图片本身没有类型，同一张图可以有不同用途
- 上传时不需要选择图片类型
- **左键**：选中图片（预览/高亮）
- **右键菜单决定用途**：「设为当前场景背景」/「设为当前战术地图」/「Showcase 给玩家」/「重命名」/「删除」
- 可选标签筛选（用户可自行给图片打标签，不强制）
- 支持系统预设背景图（内置一些通用氛围图，GM 不需要上传就能快速创建场景）

### ⑦ 聊天面板（修复对齐）

- **位置**：`fixed bottom-3 right-4`
- **现有功能保持**：消息列表、输入框、骰子支持、角色/座位切换、收起/展开
- **修复**：输入框与消息卡片宽度对齐（当前输入框比消息卡片更宽）

### 右侧组件宽度对齐

团队面板（③）和聊天面板（⑦）位于屏幕右侧，两者应**统一宽度**，视觉上形成对齐的右侧栏。

## 设计原则

1. **氛围感至上**：默认状态下场景占据最大视野，所有 UI 面板可收起
2. **场景 ≠ 背景图**：场景是叙事容器（关卡），背景图只是场景的一个属性
3. **图片无类型**：图片资产统一管理，用途由场景配置决定（右键菜单）
4. **GM/玩家自然分离**：GM 看到更多工具栏和管理面板，玩家界面干净简洁
5. **渐进式复杂度**：基础操作直觉可达，高级功能按需展开

## 与现有架构的兼容性

- 场景数据模型不变（`scene.atmosphereImageUrl`、`scene.battleMapUrl`、`scene.combatActive` 等）
- 实体系统不变（全局 `entities` + 场景 `entityIds` 引用）
- 需要新增：场景复制功能（深拷贝 scene 数据 + 生成新 UUID）
- 需要新增：图片资产的标签系统（可选，低优先级）
- 需要新增：预设背景图（内置资源，低优先级）

## 验证方式

1. 启动开发服务器，以 GM 身份进入房间
2. 验证场景按钮（左下）弹出大图卡片式场景列表
3. 验证 GM 工具栏（底部居中）的 4 Tab + 战斗按钮
4. 验证所有面板的收起/展开功能
5. 以玩家身份进入，验证看不到场景按钮和 GM 工具栏
6. 验证图库 Tab 的右键菜单（设为背景/设为地图）
7. 收起所有面板后，验证全屏场景沉浸效果
