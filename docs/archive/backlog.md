# 潜在方向

> 开发过程中发现的待探索问题和改进方向。这些不是已确认的开发计划，而是未来可能需要关注的事项。

## 测试覆盖

- **多客户端同步测试不足**：当前自动化测试全部为单客户端单元测试。作为多人协作应用，核心场景（多用户选座、场景同步、token 协同操作）缺少多 Y.Doc 同步测试。`useWorld` 竞态 bug 就是因此漏过的。需要系统性补充双客户端 `Y.applyUpdate` 测试。
  - 发现于：2026-03-10，排查 useWorld 嵌套 Y.Map 竞态条件时

## 架构

- **Y.Doc 数据迁移**：当前数据结构变更后需要手动清除 `db/` 目录。如果未来有持久化用户数据的需求，需要考虑数据迁移方案（版本号 + 迁移脚本）。

## 性能

- **大房间场景**：架构评审（`docs/architecture/system-review-2026-03-09.md`）指出当前架构适合 4-10 人小型私人游戏，更大规模场景存在瓶颈。
