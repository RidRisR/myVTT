# 数据层与 UI 层的交互模型

> 本文描述插件组件（UI 层）与数据/逻辑层之间的交互架构。
> 目标：让插件开发者理解"数据怎么进来、操作怎么出去"。

---

## 一、核心约束

在进入具体通路之前，先确立三条不可打破的规则：

1. **组件只能读数据，不能直接写**
   `sdk.data` 是只读视图。任何修改都必须通过 Workflow 提交。

2. **Workflow 不知道 UI 长什么样**
   Workflow 只处理数据和规则逻辑。它通过 Event Bus 发出"发生了什么"的信号，而不是直接调用某个组件的方法。

3. **`interaction` 只是触发入口**
   DnD、布局拖动等交互原语本身不持有业务状态——它们最终都会把用户意图转交给 Workflow。

这三条规则共同保证了插件之间的隔离性：一个插件崩溃不会影响其他插件，插件不需要知道其他插件的存在。

---

## 二、四条交互通路

### 通路 1：读取（Data → UI）

```
sdk.data.entity(id) / sdk.data.entities()
          ↓
    Component 渲染 UI
```

组件通过 `sdk.data` 主动拉取当前状态并渲染。

**响应式问题**：`sdk.data` 背后的实现决定了数据变化时组件是否自动更新。在当前 POC 中是静态 mock；在真实系统中，`IDataSDK` 由 zustand store 实现，组件通过 selector 订阅变化，数据改变时自动触发 re-render。这一细节对插件开发者完全透明——组件只需调用 `sdk.data`，更新机制由宿主负责。

---

### 通路 2：写入（UI → Workflow → Data）

```
用户操作（点击按钮、提交表单……）
          ↓
sdk.workflow.run(workflowHandle, input)
          ↓
Workflow 执行业务规则
          ↓
修改 Data Layer
          ↓
订阅了相关数据的组件自动 re-render
```

这是最基础的写入路径。组件将用户意图（输入参数）交给 Workflow，由 Workflow 决定如何修改数据。组件不关心修改是如何发生的，只关心结果。

---

### 通路 3：拖拽（UI → DnD → Workflow → Data）

```
sdk.interaction.dnd.makeDraggable({ type: 'card', data: { id } })
          ↓ 用户拖起
[HTML5 DnD 在浏览器内传递 payload]
          ↓ 用户放下
sdk.interaction.dnd.makeDropZone({
  onDrop: (payload) => sdk.workflow.run(handle, payload)
})
          ↓
回到通路 2
```

DnD 是通路 2 的**触发入口变体**。`onDrop` 回调的唯一职责是将 payload 转交给 Workflow；之后的流程与普通点击完全一致。

`sdk.interaction` 作为整体在 play 模式下注入、edit 模式下不注入，确保编辑状态下系统浮层能接管所有鼠标事件，组件无需处理模式切换。

---

### 通路 4：副作用（Workflow → UI，单向）

```
Workflow 执行完毕
          ↓
ctx.events.emit(handle, payload)
          ↓
[Event Bus 广播]
          ↓
订阅了此事件的组件响应（播放动画、弹 Toast……）
```

这条路径是**单向的、瞬时的**。Workflow 只管发出语义信号（"命中了"、"法术施放了"），订阅方自行决定如何表现。信号是"过时不候"的——错过就算了，不影响持久状态。

> **为什么不用回调？**
> 早期设计里 `WorkflowContext` 有 `showToast()`、`playAnimation()` 等硬编码回调。这要求宿主提前知道"所有可能的 UI 副作用"，插件无法扩展自定义效果。Event Bus 将这个决定权交还给组件层。

## 三、全局数据流

```
┌─────────────────────────────────────────────────────────┐
│                     Plugin Component                    │
│                                                         │
│  ① sdk.data.entity()      ←── 读取当前状态             │
│  ② sdk.workflow.run()     ──→ 提交用户操作              │
│  ③ sdk.interaction.dnd    ──→ 拖拽触发 workflow.run()   │
│  ④ sdk.events.useEvent()  ←── 监听 Workflow 副作用      │
└─────────────────────────────────────────────────────────┘
        ② run()  ③ run()              ④ emit()
┌─────────────────────────────────────────────────────────┐
│                    Workflow Engine                      │
│   读取 input → 执行业务规则 → 写入 Data → emit events   │
└─────────────────────────────────────────────────────────┘
        写入 ↓                         读取 ↑
┌─────────────────────────────────────────────────────────┐
│                      Data Layer                         │
│         Entity / Scene / Token  （zustand store）       │
└─────────────────────────────────────────────────────────┘
```

## 四、一个完整场景：玩家施放法术

以"角色卡组件上点击法术按钮，命中画布上的 Token"为例，串联四条通路：

```
① 组件读取数据
  const entity = sdk.data.entity(props.entityId)
  // → 渲染角色卡，显示法术列表

② 用户点击"火球术"按钮
  sdk.workflow.run(castSpellHandle, {
    spellId: 'fire-bolt',
    casterId: props.entityId,
    targetId: selectedTargetId,
  })

③ Workflow 执行
  - 计算命中判定
  - 计算伤害
  - 修改 target 的 HP（写入 Data Layer）
  - ctx.events.emit(combatHitEvent, { targetId, damage, critical })

④ 响应副作用
  - 角色卡组件：监听 combatHitEvent，播放"受击闪红"动画
  - Toast 管理器：监听 combatHitEvent，弹出"造成 12 点伤害"提示
  - 目标 HP 已更新 → 订阅了该数据的所有组件自动 re-render
```

## 五、Session State（计划中）

上述四条通路处理的都是"游戏世界数据"（HP、坐标等）。但 VTT 还有一类数据：**当前选中了哪个 Token、激活的是哪个工具**——这些是 UI 交互状态，不属于游戏规则，但多个组件需要共享。

Event Bus 不适合这类数据（迟到者问题、多重事实来源）。计划中的 `sdk.session` 将补充第五条通路：

```
sdk.session.dispatch({ type: 'SELECT', id: 'token-123' })
          ↓
sdk.session.useSelector(s => s.selectedIds)
          ↓
[zustand store 通知所有订阅者同步更新]
```

与通路 1 的区别：`sdk.data` 是游戏世界数据（需要同步给所有客户端），`sdk.session` 是本地 UI 状态（只在当前浏览器有效）。

---

## 六、设计边界总结

| 场景                       | 正确通路                                               |
| -------------------------- | ------------------------------------------------------ |
| 读取 Entity/Token 数据渲染 | `sdk.data`（通路 1）                                   |
| 按钮点击触发业务操作       | `sdk.workflow.run()`（通路 2）                         |
| 拖拽物品转移/施法          | `sdk.interaction.dnd` → `sdk.workflow.run()`（通路 3） |
| 播放受击动画、弹 Toast     | `sdk.events.useEvent()`（通路 4）                      |
| 当前选中 Token / 激活工具  | `sdk.session`（计划中）                                |
| 组件直接修改其他组件       | **禁止**                                               |
| Workflow 调用组件方法      | **禁止**                                               |
